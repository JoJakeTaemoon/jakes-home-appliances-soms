## 1. 로그인

- URL: `https://<도메인>/ko/mobile/login` (현장 직원 전용)

### 비밀번호 분실 시
사무실에 문의 → MANAGER 또는 ADMIN 이 비밀번호 리셋 → SMS 로 임시 비밀번호 발송 → 첫 로그인 시 새 비밀번호 강제 설정.

## 2. 모바일 메뉴

- **오늘**: 오늘 배정된 방문 목록 (`/ko/mobile/today`)
- **예정**: 향후 7일 방문 (`/ko/mobile/upcoming`)
- **내 정보**: 프로필 + 로그아웃 (`/ko/mobile/profile`)

## 3. 방문 완료 위자드 (PR #13 신규)

방문 진입 → "방문 완료" 버튼 → 5-step 위자드:

1. **작업 내역**: 발견사항, 교체 부품
2. **소모품 로그**: 교체/청소한 소모품
3. **현장 사진**: 작업 사진 (최대 40장)
4. **고객 서명**: 종이/화면에 서명 사진 + 청구 + 수금
5. **검토 + 제출**

### 4단계 (청구 + 수금) — PR #13 신규 흐름

#### 청구 금액 (chargedAmount)
- 기본값: 사무실에서 미리 등록한 예상 금액 (visit.expectedAmount) 자동 prefill
- **현장에서 다른 금액으로 변경 가능**: 추가 부품, 부분 수리, 영업 할인 등
- 변경 시 자동으로 **"변경 사유" 입력란 노출 + 필수** (최소 5자)

#### 변경 사유 작성 가이드
구체적이고 짧게:
- ✅ "추가 부품 교체 (필터 2개 추가)"
- ✅ "부분 수리 (인지된 결함 일부만)"
- ✅ "영업 할인 (장기 고객 우대 -200K)"
- ❌ "확인 필요"
- ❌ "테스트"
- ❌ "?"

사유는 매니저가 감사 로그에서 확인합니다. 모호한 사유는 후속 문의 대상.

#### 수금 금액 (collectedAmount)
실제 고객에게 받은 금액. 청구 금액과 같지 않을 수도 있음 (현장에서 부분만 수금 등). 차액은 자동으로 **잔여 (carryover)** 처리되어 다음 차수로 이월.

#### 결제 방법
CASH / BANK_TRANSFER / CARD / OTHER 중 선택. 영수증 PDF 에 한국어 + 베트남어 병기 표시 (e.g. "은행 이체 / Chuyển khoản").

### 영수증 PDF (PR #11 신규 레이아웃)
- A4 한 장에 **동일 영수증 2부** + 가운데 절취선
- 상단/하단 절반 모두 같은 내용 — 가위 그림 따라 자른 후 한 부는 고객 보관, 한 부는 회사 보관
- 금액 박스에 ₫ 단위만 인쇄되어 있음 → **기사가 수금 후 펜으로 실제 수금액 손기입**
- 영수증 하단에 서명선 2개 (지급인 / 수취인) — 종이 위 실제 서명

### 청구 금액이 다를 경우 발생하는 것
1. visit.expectedAmount 가 새 값으로 갱신
2. `VISIT_CHARGE_OVERRIDE` 감사 로그 자동 생성 (사무실에서 확인 가능)
3. 영수증 PDF 의 expected 컬럼이 새 chargedAmount 로 표시
4. Payment 행의 expectedAmount = 새 chargedAmount
5. 잔여 (carryover) 계산도 새 값 기반

## 4. 오프라인 대응

신호가 약한 현장에서는 위자드를 작성한 후 "제출" 버튼이 자동으로 큐에 저장. 신호 회복 시 자동 전송.

## 5. 본사 호출

모바일 우측 상단의 "본사 호출" 버튼 → 회사 등록 번호로 전화 자동 연결. ADMIN/MANAGER 가 "회사 정보" 페이지에서 변경한 번호가 60초 안에 반영됨.

# 직원(STAFF) 매뉴얼 — Seoul Aqua SOMS

> 본 매뉴얼은 2026-06-01 sprint 기준. STAFF 는 본사 직원 그룹 중 가장 기본 권한이며, 일상 운영을 담당합니다.

## 1. 로그인

ADMIN/MANAGER 와 동일 — `https://<도메인>/ko/login`.

## 2. STAFF 가 볼 수 없는 메뉴

- **관리자** 그룹 전체 (감사 로그, 제품 카탈로그, 사용자 관리, 회사 정보, 알림 서식) — 사이드바에 표시되지 않음 (PR #10 데이터 기반 필터)

STAFF 가 직접 URL 입력으로 진입 시도 시 "관리자 또는 매니저 권한이 필요합니다" 화면.

## 3. STAFF 작업 영역

- **고객**: 등록, 수정, 사이트(B2B) 관리, 담당자 관리
- **장비**: 설치, 이동, 교체, 상태 변경
- **계약**: 작성, 상태 전환 (DRAFT → PENDING → SIGNED → ACTIVE)
- **방문**: 일정 조정, 기사 배정, 취소
- **서비스 요청**: 검토, 승인/거부, 일정 잡기
- **수금**: 기사가 수금한 내역 모니터링, 은행 입금 등록, 사후 화해(reconcile)
- **세금계산서**: B2B 고객의 e-tax invoice 업로드 + 이메일 발송
- **보고서**: 대시보드, 통계, CSV 내보내기

## 4. 기사 청구금액 변경 모니터링 (참고)

기사가 visit-complete 시 청구금액을 변경한 경우 감사 로그에 `VISIT_CHARGE_OVERRIDE` 가 기록됩니다. 그러나 STAFF 는 감사 로그 접근 권한이 없으므로, **MANAGER 가 모니터링** 합니다. 의심스러운 case 는 MANAGER 에게 보고.

대신 STAFF 는 다음을 확인 가능:
- 방문 상세 페이지 → "비고" 영역 에 기사 입력 내용
- 수금 페이지 → Payment 행의 expectedAmount 가 사전 등록 값과 다르면 변경됨을 추정

## 5. 알려진 제약

- 사이드바에 관리자 그룹이 안 보임 — 권한 안내 일관성 위해 STAFF 가 직접 URL 입력 진입 시도해도 not-allowed 차단
- 가격 변경, 사용자 관리, 카탈로그 수정 등은 MANAGER+ 권한 필요
- URL prefix 재배치 (phase 2) 시 본사 페이지가 `/ko/o/*` 로 이동 예정
